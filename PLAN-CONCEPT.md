# Plan: Реализация концепции «Лютые Пляжники» в IPT-main

## Context
Пользователь предоставил концепцию турнира «Лютые Пляжники» и образец Excel-файла (ТАЙСКАЯ 17.03.xlsx). Нужно привести PWA-приложение в соответствие с описанной механикой: система очков за качество победы, коэффициент эффективности, zero-sum валидация, правильный посев во 2-й тур, номинации.

**Образец турнира**: 32 человека (16М + 16Ж), 4 группы Phase 1, 4 финала Phase 2 (HARD/ADVANCE/MEDIUM/LIGHT).

---

## Этап 1: Система очков за качество победы (Очки ⭐)

**Что**: Победы оцениваются по разнице мячей, а не просто как 1/0.
- Разница ≥7 → 3 очка 🔥
- Разница 3–6 → 2 очка 💪
- Разница 1–2 → 1 очко 👍
- Поражение → 0 очков

**Файлы**:
- `assets/js/ui/ipt-format.js` → `calcIPTGroupStandings()` (строки 175-200)
  - Добавить поле `qualityPts` в stats объект
  - Формула: `if (done && s1 > s2) { const margin = s1-s2; qp = margin>=7 ? 3 : margin>=3 ? 2 : 1; }`
- `assets/js/screens/ipt.js` → `_renderIPTCrossTable()` (строки 223-289)
  - Добавить колонку «Оч⭐» между «WR» и «±»
- `assets/js/ui/stats-recalc.js` → `recalcAllPlayerStats()` (строки 37-75)
  - Добавить `iptQualityPts` в wallet

---

## Этап 2: Коэффициент эффективности (Кэф ⚡)

**Что**: K = (60 + Итого) / (60 − Итого), где Итого = суммарная разница мячей.
- K > 2.0 → Абсолютная доминация
- K ∈ [1.3, 2.0] → Хороший результат
- K < 0.9 → Спад в игре

**Файлы**:
- `assets/js/ui/ipt-format.js` → `calcIPTGroupStandings()`
  - Добавить `kef: (60 + s.diff) / (60 - s.diff)` в standings
- `assets/js/screens/ipt.js` → `_renderIPTCrossTable()`
  - Добавить колонку «Кэф⚡»
- `assets/js/ui/ipt-format.js` → `finishIPT()`
  - Добавить `kef` в `iptStats` объект winners

---

## Этап 3: Обновление тайбрейка (сортировка standings)

**Текущий порядок**: wins → diff → pts
**Правильный порядок**: qualityPts → diff → pts (набранные мячи)

**Файл**: `assets/js/ui/ipt-format.js` → `calcIPTGroupStandings()` (строки 196-199)
```
.sort((a,b) =>
  b.qualityPts !== a.qualityPts ? b.qualityPts - a.qualityPts :
  b.diff !== a.diff ? b.diff - a.diff :
  b.pts - a.pts)
```

## Этап 4: Zero-Sum валидация

**Что**: Сумма разниц всех 8 игроков на корте = 0. Если пара выиграла 15:10, +5 победителям, −5 проигравшим.

**Файлы**:
- `assets/js/screens/ipt.js` → `renderIPTGroup()` / `_renderIPTCrossTable()`
  - Показать строку "∑ = 0 ← баланс" внизу таблицы
  - Зелёный если 0, красный если != 0 (индикатор ошибки ввода)
- Логика: `group.rounds.forEach(r => r.courts.forEach(c => sum += (c.score1-c.score2)*2))`
  - Каждый матч: team1 получает +(s1-s2), team2 получает -(s1-s2) → всегда 0 по определению
  - Реально проверять: ∑ score1 + ∑ score2 = const (все мячи сыграны)

## Этап 5: Рандомайзер (жеребьёвка)

**Текущее**: `generateIPTGroups()` берёт участников последовательно (slice).
**Нужно**: Случайное распределение (shuffle перед разбивкой на группы).

**Файл**: `assets/js/ui/ipt-format.js` → `generateIPTGroups()`
- Добавить Fisher-Yates shuffle перед slice: `const shuffled = [...participants].sort(() => Math.random() - 0.5)`
- Опционально: кнопка "Перемешать" в UI для ре-жеребьёвки

## Этап 6: Посев во 2-й тур (распределение по зонам)

**Текущий алгоритм**: block partition по рангу (rank / ranksPerBucket)
**Алгоритм из концепции** (для 32 игроков, 4 группы, раздельно М/Ж):

| Зона | Критерий |
|------|----------|
| 🔴 HARD | 1-е место ×3 групп + лучшее 2-е место |
| 🟣 ADVANCE | Вся Группа 4 (целиком) |
| 🟠 MEDIUM | 2 оставшихся 2-х + 2 лучших 3-х |
| 🔵 LIGHT | Худшее 3-е + все 4-е места |

**Решение**: Реализовать как специальный алгоритм для 4-group format. Для 2-group format оставить текущий (ХАРД/ЛАЙТ). Для 3 групп — промежуточный вариант.

**Файл**: `assets/js/ui/ipt-format.js` → `generateIPTPhase2Groups()`
- Новая логика посева с сортировкой кросс-группового рейтинга (Очки⭐ → Разница → Набранные)

## Этап 7: Номинации (автоматические награды)

**Что**: После `finishIPT()` вычислять спецноминации:
1. 🔥 **Лютый режим** — максимум очков⭐ + разница за весь турнир (R1+R2)
2. ⚡ **Камбэк века** — максимальный прирост между R1 и R2
3. 🌪 **Разрыв зала** — наибольший рост в общем рейтинге (из аутсайдеров R1 → чемпион R2)
4. 👑 **Снайпер тура** — наибольшее абсолютное количество набранных мячей
5. 😤 **Обидный провал** — лидер R1, сдавший позиции в R2

**Файлы**:
- НОВЫЙ: `assets/js/ui/nominations.js` — расчёт номинаций
  - `function calcNominations(trn)` → `[{ id, title, icon, playerId, value, description }]`
